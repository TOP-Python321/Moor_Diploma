1. Введение:

   - Цель проекта: Разработка и создание веб-сайта, который поможет пользователям подобрать и приобрести бу автозапчасти и продать свой автомобиль в любом состоянии.
   - Основные функциональные требования: 
   - Подбор запчастей по марке, модели и году выпуска автомобиля.
   - Отображение каталога доступных бу запчастей.
   - Поиск по наличию запчастей.
   - Продажа а/м.
   - Возможность добавления выбранных запчастей в корзину.
   - Оформление заказа с указанием контактной информации и доставки.
   - Личный кабинет пользователя для просмотра истории заказов.

2. Требования к дизайну:
   - Привлекательный и интуитивно понятный пользовательский интерфейс.
   - Адаптивный дизайн для корректного отображения на различных устройствах (мобильных телефонах, планшетах, настольных компьютерах).
   - Четкая навигация по сайту и удобный поиск запчастей.

3. Требования к функциональности:
   - Регистрация и аутентификация пользователей.
   - Возможность просмотра информации о запчастях, такой как название, описание, изображения, наличие и цена.
   - Фильтрация запчастей по различным параметрам (марка, модель, год выпуска).
   - Добавление выбранных запчастей в корзину.
   - Оформление заказа с указанием информации о доставке и контактных данных пользователя.
   - (Опционально) История заказов в личном кабинете пользователя.
   - Административная панель для управления каталогом запчастей, управления заказами, пользовательскими аккаунтами и другими аспектами сайта.

4. Технические требования:
   - Использование современных веб-технологий и фреймворков (например, HTML5, CSS3, JavaScript, Python, MySQL, Django).
   - Обеспечение безопасности данных пользователей с помощью шифрования и других мер защиты.
   - Создание удобной административной панели для управления контентом и функциональностью сайта.
   - (Опционально) Интеграция с различными платежными системами для онлайн-оплаты заказов.
   - Возможность добавления и обновления каталога запчастей через административную панель.
   - (Опционально) Реализация поисковой оптимизации (SEO) для улучшения видимости сайта в поисковых системах.

5. Дополнительные требования:
   - Возможность оставлять отзывы и оценки на преобретенные запчасти.

6. Этапы проекта:
   - Анализ требований и составление спецификации.
   - Разработка дизайна пользовательского интерфейса.
   - Разработка базы данных и серверной части.
   - Разработка клиентской части и интерфейсов.
   - Тестирование, отладка и устранение ошибок.
   - Запуск сайта и релиз в продакшн.
   - Поддержка, обновление и дальнейшее развитие сайта.


------------------------------------------------------------------------------- Функциональность сайта -----------------------------------------------------------------------------

1. Группы пользователей и их права:

   1.1. "Гость". Неавторизованный пользователь:
      - Просмотр страницы товаров и каталогов.
      - Поиск по сайту.
      - Просмотр информации о товаре.

   2.2. Зарегистрированный пользователь:
      - Все права гостя.
      - Создание и управление аккаунтом.
      - Добавление товаров в корзину.
      - Оформление заказа.
      - Просмотр истории заказов.

   3.3. Администратор:
      - Все права зарегистрированного пользователя.
      - Управление каталогом товаров (добавление, редактирование, удаление).
      - Управление пользователями (создание, блокировка, удаление).
      - Управление заказами (обработка, отмена, изменение статуса).

2. Страницы, доступные для различных пользователей:

   2.1. Для неавторизованных пользователей:
      - Главная страница:
         - Общая информация о сервисе

      - Каталог запчастей:
         - Полный список всех доступных запчастей с фильтрами по категориям, маркам автомобилей и т.д.

      - Детальная страница товара:
         - Подробная информация о конкретной запчасти, включая описание, изображения, цену и наличие.

      - Страница контактов и обратной связи:
         - Содержит номера контактных телефонов и других способов связи.

      - Выкуп авто в разбор:
         - Содержит краткую информацию о условиях выкупа и контактную информацию.

   2.2. Для зарегистрированных пользователей:
      - Все страницы доступные неавторизованным пользователям.

      - Личный кабинет:
         - Содержит персональную информацию, историю заказов.

      - Корзина:
         - Содержит выбранные пользователем запчасти перед оформлением заказа.
         - Позволяет просмотреть, удалить или добавить запчасти в покупку.

      - Оформление заказа:
         - Страница для ввода информации о доставке и оплате при оформлении заказа.

   2.3. Для администраторов:
      - Все страницы доступные зарегистрированным пользователям,

      - Панель администратора:
         - Доступна только администраторам или уполномоченным сотрудникам.
         - Позволяет управлять каталогом запчастей, управлять заказами, управлять пользователями.

   3. Поиск запчастей:

      - Форма для поиска запчасти по номеру:
         - Если запчасть с указанным номером есть в каталоге - вывод всех доступных запчастей.

      - Каталог со всеми доступными марками автомобилей:
         - При выборе необходимой марки автомобиля будут доступны все модели и поколения а/м.
         - При выборе модели будут доступны все запчасти текущего а/м.
         - Фильтрация по категориям запчастей (детали кузова, двигатели, навесное оборудование и т.д.)

   4. Выкуп автомобиля:

      - Клиент, посредством связи с администратором через доступный мессенджер (Telegram, Viber, WhatsApp), отправляет подробные фото автомобиля, желаемую цену выкупа и контактные данные.

   5. База данных:

      - Используется БД MySQL

      Таблица "spares":
         - id: уникальный идентификатор запчасти (первичный ключ)
         - name: название запчасти
         - brand_id: id внешний ключ на таблицу "brands" (связь с таблицей brands)
         - model_id: id внешний ключ на таблицу "models" (связь с таблицей models)
         - vendor_code: артикул товара
         - state: состояние запчасти
         - описание: описание запчасти
         - detal_number: номер запчасти
         - price: цена запчасти
         - photo: фото запчасти

      Таблица "brands":
         - id: уникальный идентификатор марки машины (первичный ключ)
         - brand_name: название марки машины

      Таблица "models":
         - id: уникальный идентификатор модели машины (первичный ключ)
         - model_name: название модели машины
         - brand_id: внешний ключ на таблицу "brands" (связь с таблицей brands)
         - year: года изготовления (поколение)

      Таблица "users":
         - id: уникальный идентификатор пользователя (первичный ключ)
         - name: имя пользователя
         - email: электронная почта пользователя
         - password: зашифрованный пароль пользователя

      Таблица "orders":
         - id: уникальный идентификатор заказа (первичный ключ)
         - user: внешний ключ на таблицу "users" (связь с таблицей users)
         - spare: название запчасти
         - date: дата заказа
